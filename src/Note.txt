API
-클라이언트 - 서버 간의 약속
    클라이언트가 정한대로 서버에게 요청(Request)을 보내면,
    서버가 요구사항을 처리하여 응답(Response)을 반환

REST
-주소에 명사, 요청 방식에 동사를 사용함으로써 의도를 명확히 드러냄을 의미
    *동사 : JPA 시간에 배운 CRUD를 지칭
        즉, A에 대해 생성(POST)/조회(GET)/수정(PUT)/삭제(DELETE)요청을 하는 것
        예시)
            *GET/course
                -> 강의 전체 목록 조회를 요청

            *GET/course/1
                -> ID가 1번인 데이터 조회 요청

            *POST/course
                -> 강의 생성 요청

            *POST/course/3
                -> ID가 3번인 데이터 수정 요청

            *DELETE/course/2
                -> ID가 2번인 데이터 삭제 요청

        ** 주의사항
            * 주소에 들어가는 명사들은 복수형을 사용
                /course

            * 주소에 동사는 가급적 사용하지 않음
                /account/edit


Put이 잘 작동되는지 확인하는 방법
    1. GET으로 저장한 데이터가 잘 돌아오는지 확인
    2. 돌아온 데이터를 변경
    3. 변경된 데이터가 제대로 변경 되었는지 확인
